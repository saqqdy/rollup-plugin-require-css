Code Splitting Example

This demonstrates CSS code splitting for multi-entry builds. Each entry point gets its own CSS bundle.

Build Output

dist/
├── app.js      + app.css    (App entry)
└── admin.js    + admin.css  (Admin entry)

Click the links below to see each entry point with its own CSS bundle.

📱 App Entry

Loads app.css only

⚙️ Admin Entry

Loads admin.css only

Configuration

// rollup.config.js
export default {
  input: ['src/app.js', 'src/admin.js'],
  output: { dir: 'dist', format: 'es' },
  plugins: [
    requireCss({
      split: true,
      output: (chunk) => `${chunk.name}.css`
    })
  ]
}
